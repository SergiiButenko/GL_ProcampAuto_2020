ЗАДАЧА:
"""
У вас есть два ветки: master, US_123, которая растет от master. В master нашли багу несовместимую с жизнью продукта.
1. Вам срочно нужно сделать хотфикс баги, которая была найдена в ветке master.
2. После того, как вы закоммититли и зарелизили багу, вам нужно получить этот же фикс в ветке US_123
"""

ВАРИАНТ РЕШЕНИЯ:

Создаем хотфикс:
1. Находясь в master, создадим ветку для хотфикса и перейдем в нее: git checkout -b hot_fix
2. Пушим ветку в remote: git push --set-upstream origin hot_fix
3. Пишем хотфикс, добавляем в индекс, коммитим, пушим в remote:
4.  git add . (или то, что нужно добавить)
    git commit -m "hot fix resolving the problem <blah-blah-blah>"
    git push origin
5. На GitHub создаем Pull Request на слияние с master
6. После PR confirmation, локально переходим на ветку US_123:
    git checkout US_123
7. Стягиваем содержимое ветки master с хотфиксом в локальную ветку:
    git pull origin master
